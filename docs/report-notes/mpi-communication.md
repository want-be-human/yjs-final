# MPI 通信模式示意

数值积分选的是梯形法，把 `[0, 1]` 切成固定数量的小区间。基础 MPI 版的通信关系如下：

```text
rank 0
  |
  | scatter: 分发每个进程负责的起点和区间数量
  v
rank 0     rank 1     rank 2     rank 3
  |          |          |          |
  | 本地累加 | 本地累加 | 本地累加 | 本地累加
  |          |          |          |
  +----------+----------+----------+
             reduce: partial sum 求和回 rank 0
```

非阻塞版本没有用 `scatter/reduce` 的集合通信，而是由 rank 0 先用 `Isend` 发任务，再用 `Irecv` 提前挂起结果接收。这样 rank 0 在等待 worker 返回之前可以先计算自己的分片。这个例子里每个进程的计算量远大于通信量，所以提升不一定明显，主要用于体现通信改写方式。
